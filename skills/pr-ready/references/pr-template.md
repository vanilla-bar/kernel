# PRテンプレート

## 本文テンプレート

```markdown
## 概要
<何をなぜ変えたか>

Closes #xxx（既存のリンクがあれば保持）

## 関連ドキュメント
<docs/ 配下に追加・変更したドキュメントがあれば、ファイルパスと要点を記載する>
<ドキュメントがない場合はこのセクションを省略する>

## 変更内容
- <コミットベースの箇条書き>

## 確認したこと
- [ ] lint通過（npm run lint）
- [ ] ビルド通過（npm run build）
- [ ] 画面で動作確認済み
- [ ] ドキュメントリント通過（npm run lint:docs）
```

## ドキュメント記載のポイント

- `docs/` に変更がある場合、レビュアーがドキュメントを読めば実装意図を把握できるよう、何を決めて何を書いたかを丁寧に説明する
- ドキュメントと実装は一致している前提なので、ドキュメントの説明が充実していればレビューの負荷が下がる

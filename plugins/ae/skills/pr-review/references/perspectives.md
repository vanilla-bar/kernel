# PRレビュー観点

PRの差分に対して、以下の観点でレビューする。
プロジェクトの `.agents/pr-review-perspectives.md` で上書きカスタマイズ可能。

## コード品質

| 観点 | チェック内容 |
|------|-------------|
| 既存パターンとの一貫性 | 命名規則・ディレクトリ構成・コードパターンが既存コードと一致しているか |
| 不要なコード | デバッグ用コード、コメントアウト、未使用のimport/変数が残っていないか |
| 複雑度 | 不必要に複雑な実装がないか。もっと単純な方法がないか |
| エラーハンドリング | エラーケースが適切に処理されているか |

## ロジック・正確性

| 観点 | チェック内容 |
|------|-------------|
| 境界値・エッジケース | null/undefined、空配列、0、負数など境界値の考慮漏れがないか |
| 副作用・リグレッション | 変更が意図しない箇所に影響していないか |
| データ整合性 | スキーマ定義やAPI仕様と矛盾がないか |
| 条件分岐の網羅性 | if/switch文で漏れているケースがないか |

## セキュリティ

| 観点 | チェック内容 |
|------|-------------|
| 入力バリデーション | ユーザー入力が適切に検証されているか |
| 認証・認可 | 適切なアクセス制御がされているか |
| 機密情報の露出 | APIキー、パスワード、個人情報がハードコードやログに露出していないか |

## 変更の整合性

| 観点 | チェック内容 |
|------|-------------|
| PR説明との一致 | 差分がPRの説明（タイトル・本文）と一致しているか |
| 影響範囲の見落とし | この変更に伴って更新すべきだが漏れているファイルがないか（型定義、テスト、ドキュメント等） |
| スコープの適切さ | PRの目的と無関係な変更が混ざっていないか |
